## 职位描述

业务介绍

我们在做具身智能的数据基础设施。具体来说，是为任何机器人场景构建从"数据采集 → 清洗标注 → 增强合成 → 策略训练 → 部署验证"的完整数据闭环。当前具身智能的核心瓶颈不在模型架构（Diffusion Policy、ACT、VLA、WAM 等方法已经证明了可行性）；瓶颈在数据，高质量的操作轨迹数据极度稀缺，获取成本高，格式碎片化，跨机器人复用困难。我们要解决的就是这个问题。

当前重点方向有低成本数据采集、多源数据统一管理、仿真数据大规模生成、数据质量驱动策略上限。

团队处于早期建设阶段，你将成为从零定义"具身数据应该怎么做"的核心角色。

岗位职责

1、设计多源异构数据采集管线：对接遥操作、人手动捕、外骨骼、仿真等不同数据源，完成从原始传感器数据到可训练格式的全链路

2、构建数据处理与增强算法：视觉域适配（人手→机器人手 inpainting）、异常轨迹检测与清洗、多模态对齐（RGB + 深度 + 触觉 + 关节角 + 力矩）

3、搭建具身数据平台：统一数据格式与版本管理、元数据索引、数据可视化回放、训练数据高性能加载

4、实现大规模仿真数据生成：分布式调度仿真环境，自动标注与轨迹过滤，难度递增采样

5、跟踪数据方向前沿并落地：将学术界最新的数据采集范式、跨具身迁移方法、数据扩增策略转化为可用的工程方案

## 任职要求

任职要求

1、计算机科学、机器人学、人工智能等相关专业本科及以上学历

2、扎实的 Python 工程能力，熟悉 PyTorch 生态

3、熟悉至少一种：分布式计算（Ray / Spark）、大规模数据处理（Arrow / Parquet / HDF5）、对象存储（S3 / OBS）

4、有实际的数据管线开发经验（数据采集、清洗、存储、加载）

5、良好的系统设计能力，能独立设计和落地数据平台

加分项

1、有机器人学背景或操作数据处理经验（遥操作、动捕、仿真数据）

2、熟悉 LeRobot / ManiSkill / MuJoCo / Isaac Lab 等机器人学习框架

3、了解 Diffusion Policy / ACT 等模仿学习算法的数据需求

4、有 NeRF / 3DGS / 视觉 inpainting 等 3D 视觉经验

5、有 Docker / K8s 容器化部署经验

6、在顶会（CoRL / RSS / ICRA / NeurIPS / CVPR）发表过相关论文
