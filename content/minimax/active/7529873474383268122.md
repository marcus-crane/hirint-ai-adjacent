## 职位描述

一、为什么加入我们

1. 离模型最近的数据工程：你的工作直接影响模型训练数据质量，和算法团队深度协作；

2. AI First 文化：我们用 AI 重新定义开发范式，鼓励工程师探索 AI 工具与新范式；

3. 真实的大规模场景：千亿级数据量下的工程挑战，成长曲线远超普通业务后端；

4. 扁平高效的团队：快速决策，想法能直接落地。

二、我们在做什么

MiniMax 正在构建驱动下一代 AGI的数据基础设施。大模型的能力边界，很大程度上由数据决定——我们的团队直接参与数据采集、清洗、处理全链路的工程建设，和算法团队并肩定义模型的数据供给体系。

这不是普通的 CRUD 后端，而是在为 SOTA 模型持续"供粮"的核心平台。你将深度参与分布式爬虫调度、大规模数据管道、存储与解析系统的设计与实现，在真实的 AGI工程场景中快速成长。

三、你将做什么

1. 参与 AGI数据平台后端系统的设计与研发，构建高可用、高扩展的数据采集与处理基础设施；

2. 参与分布式爬虫调度系统、数据存储与处理管道核心模块的开发，提升系统吞吐量与资源利用率；

3. 参与数据采集、清洗、存储、解析全链路的技术方案设计与优化，保障系统在大规模数据场景下的稳定性；

## 任职要求

四、我们期待你

基础要求：

1. 熟练掌握 Go / Java / Python 中至少一门语言，具备 1年以上后端开发经验，有良好的代码规范意识；

2. 理解分布式系统基本原理，对高并发、大规模数据处理有一定实践或学习兴趣；

3. 了解常用数据库、缓存或消息队列等中间件的基本原理与使用；

4. 有较强的问题排查与自驱学习能力，能独立承担模块设计与实现。

加分项：

1. 对 AI / 大模型技术方向有浓厚兴趣，关注社区进展（LangChain、MCP、vllm

等）并有相应实践；

2. 有数据管道、资源调度、爬虫系统等相关开发经验；

3. 对主流开源项目（分布式存储、容器化、服务治理等）有了解或贡献；

4. 有个人 AI 相关项目或开源项目经验；
