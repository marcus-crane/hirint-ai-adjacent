## 职位描述

MiniMax 是国内头部大模型公司，开放平台是商业化的核心战场。Token 是大模型公司的"货币"——怎么让开发者用起来、怎么让企业愿意付费、怎么设计商业模式让收入规模化，这个岗位直接决定公司商业化的天花板。

职位描述：

1. 负责 AI API 网关平台的后端研发，跟进主流 LLM API 变化，持续完善 OpenAI/Anthropic 兼容接口、Responses API、Chat Completions、Embeddings、Rerank、图像/音频等接口适配能力。

2. 优化网关高并发场景下的稳定性、延迟、限流、并发控制、缓存、队列与可观测性，负责token分配、算力管理，包括权重/优先级分流、流量打标、失败重试、自动熔断、自动禁用、健康检查与故障切换。

3. 参与管理后台、开放 API、部署运维工具链建设，支持 Docker、Docker Compose、私有化部署场景，建设 Token 级用量统计、成本核算、模型倍率、用户配额、调用日志、账单与运营数据分析等B端能力。

4. 参与安全与合规设计，包括鉴权、密钥加密存储、审计日志、异常风控、上游账号授权边界等。

这个岗位不是写功能代码，是设计大模型公司的商业模式并把它工程化。你做的每一个决策，会直接影响开发者的使用体验和企业的付费意愿，最终决定开放平台能走多远。

## 任职要求

1.- 2-8 年后端研发经验，熟悉 Go、Java、Node.js、Python 中至少一种，优先 Go 或 TypeScript/Node.js。

2. 熟悉 HTTP API、RESTful、SSE/流式响应、反向代理、网关、鉴权、限流、熔断、负载均衡等基础能力。

3. 熟悉 PostgreSQL / MySQL、Redis、MQ，并能设计可靠的数据模型、缓存策略和并发控制方案。

4. 具备良好的工程习惯，能独立拆解复杂需求，关注稳定性、安全性、可维护性和长期演进。

5. 加分项：

- 深度使用或参与过 new-api、one-api、sub2api、LiteLLM、Kong、APISIX、Traefik、Nginx Gateway 等项目。

- 熟悉 OpenAI-compatible API 生态，理解不同模型厂商之间的参数差异、流式协议差异、错误码差异和兼容适配难点。
