## 职位描述

为什么加入我们

1. 用 AI 重新定义一家 AI公司的工作方式：让整个公司的运营环境 AI Native化——你定义的工具和流程，将成为数百人日常工作的底层操作系统

2. 技术决策权极大：自主选型、自主落地，ClaudeCode、Cursor、MCP、Agent 工作流——你认为最高效的方案就是方案

3. 离 AGI 产品最近的内部工程：公司本身就在做最前沿的多模态大模型，你的 AI Native实践天然能拿到最新的模型能力和内部资源

4. 扁平决策，直接影响公司：向高层直接汇报，想法能快速落地，与一流工程团队并肩作战

我们在做什么

这是一个研发驱动的内部提效角色——你需要用工程化思维和 AI工具，重构公司内部的协作流程、开发工具链和运营基础设施，让"AI是同事"从口号变成每个人的日常体感。

你将做什么

1. 主导公司内部 AI Native 工具链建设：基于 Claude Code、MCP Server、Agent

工作流等，构建覆盖研发、运营、协作的 AI 自动化体系

2. 设计并落地内部提效平台：识别公司各团队的重复性工作，用 AI +工程手段系统性消除，量化提效收益

3. 构建企业 IT 基础设施的智能化底座：网络、身份认证、办公协同、终端管理等，以AI-first 思维重新设计而非简单搬运传统方案

4. 组建并带领一支研发团队，推动 AI Native实践在全公司落地，建立内部最佳实践和标准

5. 与算力/AI Infra 团队协作，确保内部工具能充分利用公司自有模型能力

## 任职要求

我们期待你

基础要求：

1. 5 年以上研发经验，1 年以上研发团队管理经验，有明确的技术 leader 履历

2. 深度使用 AI 编程工具（Claude Code / Cursor / Copilot）进行日常开发，有AI-assisted 工程实践的真实体感和方法论

3. 有内部工具/平台/效能方向的建设经验，能从业务痛点出发设计工程方案

4. 熟悉云原生技术栈，具备全栈或偏后端的工程能力

5. 强自驱力和产品思维，能把"内部提效"当产品来做

加分项：

1. 有 AI Agent / MCP / LLM 应用开发经验，做过 AI 工作流自动化的实际项目

2. 有企业 IT 基础设施经验（网络、安全、身份认证），能兼顾传统 IT 底座建设

3. 在技术社区有 AI Native 工作方式的分享或影响力

4. 对"用 AI 替代一切不值得人做的事"有执念
