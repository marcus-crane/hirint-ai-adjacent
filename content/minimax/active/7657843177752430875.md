## 职位描述

一、一句话定位

专攻在线业务 CPU 性能的诊断与优化专家——解决线上各类资源争抢、性能瓶颈、长尾延迟问题，把在线服务的性能与资源效率做上去。

二、你将负责什么

1. 性能诊断：定位线上 CPU 相关的资源争抢（CPU / cache / 内存带宽 / 锁 / IO）、毛刺与长尾延迟、吞吐瓶颈，尤其是混部 / 高并发场景下的干扰。

2. 性能剖析与调优：用 perf / eBPF / 火焰图等做 profiling，找到内核态 / 用户态热点，给出可落地的优化（代码 / runtime / NUMA / 绑核 / 内核参数）。

3. 资源隔离与 QoS：cgroup / 容器资源隔离、在离线混部的干扰抑制、CPU 拓扑 / NUMA 亲和，保障在线业务性能不被挤占。

4. 体系沉淀：性能基线 / 压测 / 容量评估，把性能纳入 SLO 与可观测，沉淀诊断方法论与自动化工具。

## 任职要求

1. 5 年以上性能工程经验，深厚的 Linux 系统性能功底。

2. 精通性能剖析工具链（perf、eBPF / bpftrace、火焰图、ftrace 等），懂 CPU 微架构 / cache / NUMA / 调度器 / 中断。

3. 熟悉资源隔离与混部（cgroup、容器、QoS、干扰定位）。

4. coding 硬（C/C++/Go/Rust 至少一门 + 脚本）——能造诊断 / 自动化工具，非纯运维。

5. 强根因分析与稳定性方法论；能与业务研发协同把性能问题解决掉、并讲清价值。

加分

1. 内核 / 调度器 / 网络栈源码级理解；JVM 或 Go runtime 性能调优。

2. 大规模在线服务长尾延迟治理、eBPF 可观测平台建设、相关开源贡献。
