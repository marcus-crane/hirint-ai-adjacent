## 职位描述

你将负责系统性提升大模型在软件工程场景下的代码能力，覆盖前端、移动端、桌面端与服务端等方向。当前大模型已具备一定代码生成能力，但在复杂工程场景中距离「可放心交付」仍有显著差距。

瓶颈往往不在模型本身，而在如何与真实的生产力产品结合：模型要在特定方向持续变强，需要清晰定义该方向中真正有难度的任务、如何构建让模型完成任务的环境（RL environment），以及如何判定完成质量（reward）。这要求一线工程判断力——唯有亲手构建过复杂系统的人，才能识别真实问题，并设计出可抵御刷分（reward hacking）的评测。

该岗位对模型 coding 能力的发展方向、质量标准、数据与评测构建方式拥有充分定义权。

## 任职要求

岗位职责

- 定义方向——确定各端方向上模型优先解决的问题、需构建的任务、环境搭建方式与 reward 设计

- 建立评测标准——构建评测体系（eval）、定义质量标准、把控数据质量，使能力提升可量化，并确保评测可抵御 reward hacking

- 亲手实践——搭建 harness、编写 skill、运行 agent，亲自完成最高难度的数据与评测打样，而非仅做方案设计

任职要求

- 技术背景偏向高复杂度的系统方向，并具备跨端的一线开发经验（服务端、客户端、Web、桌面端，或端侧推理、音视频等方向中的若干）

- 架构 owner 级的工程深度独立主导过复杂系统或产品的核心研发，能清晰阐述关键设计决策与权衡

- AI Native，已将 agent 深度融入研发流程，能说明所构建的 harness / hook / RAG / 多 agent 方案

- 问题定义与评审能力，能独立厘清 coding agent 在特定方向的目标与数据评测方案，并为团队提供 cross-check
