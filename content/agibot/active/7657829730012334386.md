## 职位描述

职位描述:作为 操作系统（base Linux）测试工程师，您将负责公司自主研发/定制化 Linux 操作系统及核心组件的质量保障工作。您将深度参与系统从设计到发布的完整生命周期，通过构建自动化测试框架、设计严谨的测试用例以及深度的性能与稳定性调优，确保操作系统在高并发、高可靠性场景下的绝对稳定。包括不限于

1.需求分析与设计：参与操作系统相关需求的评审，理解内核、驱动、系统组件等设计架构，制定完善的测试计划与测试策略；

2.用例设计与执行：负责 Linux 操作系统底层（内核、驱动、文件系统、网络协议栈）及上层基础组件的功能、性能、可靠性、安全性和兼容性测试；

3.自动化建设：设计、开发和维护 Linux 系统自动化测试框架及工具，编写自动化测试脚本，提升测试效率与持续集成（CI/CD）水平；

4.问题定位与分析：深度追踪测试中发现的缺陷，利用系统级调试工具进行初步定位与根因分析，协同协同开发人员（或开源社区）推动问题解决；

5.专项测试：负责 OS 稳定性（如长负载、压力测试）及性能基准测试（Benchmark），输出调优建议与权威测试报告。

## 任职要求

1.计算机、电子信息、软件工程或相关专业本科及以上学历；

2.3年及以上 Linux 系统测试、内核测试或底层软件测试经验；

3.精通Linux 操作系统架构，熟悉内存管理、进程调度、虚拟化（KVM/Docker）、网络协议栈或文件系统等至少两个核心模块；

4.熟练掌握Shell / Python / Go 等至少一种脚本语言，具备独立开发自动化测试脚本或工具的能力；

5.熟悉常见的 Linux 性能测试工具（如：`fio`, `iperf`, `sysbench`, `perf`, `unixbench` 等）；

6.拥有主流 Linux 发行版（如 CentOS, Ubuntu, openEuler, Anolis OS 等）深度定制或测试经验者优先；

7.熟悉 LTP (Linux Test Project)、Autotest 等开源测试框架，或有开源社区（如 Linux Kernel）贡献经验者优先；

8.熟悉U-Boot、交叉编译及硬件接口（I2C/SPI/CAN）测试。
