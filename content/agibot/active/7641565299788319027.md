## 职位描述

1. 负责机器人MC（Motion Control，运动控制）框架的架构设计、核心模块开发与持续迭代优化，保障框架高性能、高可靠、高可扩展；

2. 主导机器人RL policy推理，运动控制逻辑、轨迹规划、关节伺服适配、实时通信等核心能力的代码实现与底层优化，支撑多机型、多场景机器人运动控制需求；

3. 负责 MC 框架的长期维护、版本管理、问题排查，定位并解决运动卡顿、时序异常、算力占用、实时性不足等底层技术问题；

4. 参与硬件（电机、驱动器、传感器）与 MC 框架的联调适配，完成驱动抽象、协议封装与控制闭环优化；

5. 以vibe coding风格进行高效高质量编码，注重代码节奏感、工程美感与可维护性，在快速迭代中保持架构清晰、逻辑优雅；

6. 编写技术文档、接口规范与示例工程，支撑上层应用、算法团队基于 MC 框架快速开发；

## 任职要求

1. 本科及以上学历，计算机、自动化、机器人、电子电气等相关专业，具备机器人控制或嵌入式开发经验；

2. 精通 C/C++ 或 Rust 等底层开发语言，熟悉实时操作系统（RTOS/Linux RT）、多线程 / 进程、锁机制与内存优化；

3. 理解机器人运动控制原理：RL policy推理，轨迹规划、插补、伺服控制、动力学模型、坐标系变换等；

4. 具备扎实的代码功底与vibe coding 能力：能在保持流畅开发节奏的同时，写出结构优雅、可读性强、鲁棒性高的代码，拒绝混乱堆砌；

5. 熟悉常见总线与通信协议（CAN/EtherCAT/Modbus/TCP/UDP），有电机驱动或机器人底盘开发经验优先；

6. 具备良好的问题定位能力，能快速分析复杂系统 Bug、性能瓶颈并给出优雅解决方案。

加分项

1. 有开源机器人项目（ROS/ROS2、MoveIt、自主控制框架）贡献或深度二次开发经验；

2. 熟悉仿真工具（Gazebo、Webots）或运动控制算法仿真验证流程；

3. 具备极强的工程审美与代码洁癖，追求简洁、高效、有节奏感的开发范式；

4. 具备跨平台开发与性能调优经验。
