## 职位描述

职位概述

我们希望寻找一位具有量产经验的集成测试工程师，深度参与下一代人工智能人形机器人产品的软件质量保障工作。你将与软件、硬件、算法、结构等跨职能团队紧密协作，构建覆盖端侧系统全链路的集成测试体系，推动产品从研发态走向量产态的质量闭环，共同打造业界领先的软件工程研发体系。

主要职责

1.负责端侧软件（嵌入式固件、驱动、中间件）、客户端软件及操作系统的集成测试全流程：

1）制定测试计划，设计覆盖功能、性能、稳定性、兼容性的测试用例；

2）使用内部集成工具完成软件部署、系统启动及测试执行，输出规范测试报告；

3）负责缺陷的发现、记录、跟踪与回归验证，推动问题闭环；

2.参与自动化测试系统建设：

1）搭建并维护 CI/CD 流水线中的自动化测试环节；

2）推动数据闭环体系建设，支撑测试结果的持续分析与迭代；

3.参与软硬件联调测试（HIL/SIL），验证传感器、电机控制、通信中间件等子系统的集成正确性；

4.设计端侧软件核心技术指标（延迟、吞吐量、稳定性等），对标行业标准，提出量化优化建议；

5.编写测试方案、缺陷分析报告及技术文档；为团队成员提供测试规范培训与指导；

6.与软件、硬件、算法、结构、产品等跨职能团队协作，确保项目按节点交付并达到量产质量标准

## 任职要求

1.本科及以上学历，计算机、软件工程、自动化、电子信息等相关专业

2.有3 年以上软件测试工作经验，其中包含嵌入式系统或机器人产品的集成测试经验

3.熟练掌握软件测试方法论（黑盒/白盒/灰盒、边界值、等价类、场景测试等），具备系统级集成测试设计能力

4.熟悉机器人系统架构，理解传感器 HAL 层、电机驱动控制层、通信中间件（如 DDS/LCM）、算法软件的基本逻辑与接口规范

5.具备 Python 或 Shell 脚本编写能力，能独立开发或维护自动化测试脚本

6.熟悉 Linux 系统操作及调试工具（gdb、strace、perf、journalctl 等）

7.熟悉 ROS/ROS2 生态工具的使用（话题、服务、录包回放等）

8.熟悉测试管理工具（JIRA、禅道、飞书项目等）及 Git 版本管理流程

9.良好的英文阅读能力，能独立阅读协议规范及开源项目文档

加分项

1.有量产阶段产品集成测试经验（SOP 制定、产线测试工装开发等）

2.熟悉 CI/CD 工具链（Jenkins、GitLab CI 等），有在流水线中集成测试环节的实践经验

3.有 HIL（Hardware-in-the-Loop）测试或仿真环境（Gazebo、IsaacSim 等）使用经验

4.有性能测试、压力测试及可靠性测试（FMEA）经验

5.了解嵌入式实时操作系统（RTOS）或 Linux BSP 基础知识

6.热爱机器人产品研发，具备产品思维，能主动从用户视角发现系统级问题。
