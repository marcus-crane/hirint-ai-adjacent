## 职位描述

主要职责

1. 测试用例设计：能基于需求文档、产品功能与客户业务场景，独立设计功能、性能、边界、异常等验收用例（含前置条件、操作步骤、预期结果、判定标准）；熟悉等价类划分、边界值分析、场景法等用例设计方法。

2. 用例执行与记录：严格依据验收用例库，在真实客户环境中执行回归测试。用例库全面覆盖功能流程、性能指标（抓取成功率、单笔订单完成时间）及各类异常处理场景（物品掉落、液体泄漏、抓取失败等）。客观、详细地记录每一项测试结果、通过与否及具体现象。

3. Bug 反馈与跟踪：使用公司指定的协作工具（飞书 / Jira），规范提交 Bug 报告。报告要素完整：清晰的问题描述、可复现的操作步骤、现场环境信息（软件版本、网络状况等），强制附加截图、录屏或短视频作为证据。持续跟踪 Bug 状态，直至研发修复并验证关闭。

4. 测试脚本编写与自动化：熟悉 Python / Shell 至少一种编程语言，能针对重复执行的验收用例编写自动化脚本（模拟订单下发、批量回归、日志采集与解析等）；了解 Selenium / Pytest / Requests 等主流自动化测试框架优先。脚本需归档到项目测试资产库，配合研发验证自动化用例有效性。

5. 验收报告编制：根据客户或项目要求，整理测试过程与结果数据，输出格式规范、内容完整的验收交付物（如《项目验收测试报告》《UAT 报告》等）。报告需逻辑清晰、数据准确，并协助交付负责人推动客户完成审阅与签字确认。

## 任职要求

1. 学历：大专及以上。

2. 专业：计算机、软件工程、电子信息、自动化、机械等相关专业。

3. 经验：1 年以上硬件或软件系统测试经验，熟悉软件测试生命周期。

4. 基础：理解测试流程与 Bug 管理基本方法，能看懂用例设计意图。

加分项

行业经验：机器人 / IoT / 智能硬件 / 自动化设备等产品的现场交付、验收或测试经验优先。

2. 机器人调试：有机器人现场测试、固件升级、参数标定经验，熟悉抓取失败、定位漂移、通信断连等常见故障复位流程。

3. 测试用例设计：能独立设计功能、性能、边界、异常等测试用例，熟悉等价类划分、边界值分析、场景法等方法论。

4. 测试脚本：熟悉 Python / Shell 至少一种，能编写自动化测试脚本（模拟订单、批量回归、日志采集与解析等）；有 Selenium / Pytest / Requests 等框架经验优先。

5. 日志阅读：能大致看懂设备运行日志中的报错信息、状态码，进行初步定位。
