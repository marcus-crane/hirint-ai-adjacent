## 职位描述

岗位背景

我们是智元机器人具身研究中心的 Infra 团队，正在为千台级机器人（Embodied AI）构建连接"物理世界"与"云端训练"的闭环数据基础设施。

本岗位负责团队核心基础设施的日常运维与稳定性保障。我们需要一位可靠的运维工程师，确保 Kubernetes 集群、中间件服务、GPU 算力平台和网络基础设施的稳定运行，让研发团队能够专注于业务创新而无后顾之忧。

核心职责

1. Kubernetes 集群运维：负责 Kubernetes 集群的日常监控、资源管理与故障排查。管理 Pod/Deployment/StatefulSet 等工作负载，处理服务异常、资源不足等常见问题，确保集群稳定运行。

2. 中间件服务运维：维护 PostgreSQL、RabbitMQ、Redis 等核心中间件的运行状态。执行日常巡检、状态监控、数据备份，处理连接异常、队列积压等运维问题。

3. GPU 算力平台运维：管理云端 GPU 计算资源，监控训练任务队列状态，处理任务调度异常，协调算力资源分配，保障大规模训练任务的顺利执行。

4. 网络与代理服务管理：维护公司互联网代理服务，确保研发人员的国际网络访问畅通；管理办公网络（WIFI）、VPN 等网络基础设施的日常运维。

5. 可观测性与告警响应：使用 Prometheus、Grafana 等监控工具进行服务状态监控，响应告警事件，快速定位问题根因，执行故障恢复操作。

6. 故障响应与值班：参与 on-call 值班轮换，对生产环境故障进行快速响应与处理，编写故障报告并推动问题根本解决。

## 任职要求

技术要求

1. 容器与编排：熟悉 Kubernetes 日常运维操作（kubectl 命令、Pod 管理、日志查看、资源监控），能够独立处理常见的集群运维问题。

2. 数据库运维：熟悉 PostgreSQL 基础运维（连接管理、慢查询排查、备份恢复），了解数据库常见问题的排查思路。

3. 消息队列：熟悉 RabbitMQ 或 Kafka 的基础运维（队列监控、消息积压处理、节点状态检查）。

4. 监控系统：熟练使用 Prometheus + Grafana 进行监控查询与仪表盘配置，能够根据监控数据定位问题。

5. 网络基础：理解 TCP/IP 基础知识，熟悉代理服务配置（HTTP/SOCKS 代理、VPN），能够排查基础网络连通性问题。

6. 脚本能力：熟悉 Python 或 Shell 脚本编写，能够编写自动化运维脚本提升工作效率。

加分项

1. 有 AI Infra 或 GPU 集群运维经验，了解训练任务调度

2. 熟悉公有云平台（阿里云、金山云、AWS 等）的运维操作

3. 有网络代理系统（Clash/Mihomo、V2Ray 等）的管理经验

4. 有 SRE、DevOps 或 ITIL 相关认证

5. 有良好的文档习惯，善于总结运维知识与故障处理经验

岗位特点

- 稳定优先：我们更看重稳定可靠的运维能力，而非追求技术深度

- 快速响应：需要具备快速定位和解决问题的能力，保障业务连续性

- 持续学习：AI Infra 领域发展迅速，需要持续学习新的运维技能

- 团队协作：与研发团队紧密配合，理解业务需求，提供运维支持
