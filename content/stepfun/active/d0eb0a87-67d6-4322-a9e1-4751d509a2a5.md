岗位职责：

1. 参与公司容器云平台核心基础设施研发，负责 Kubernetes 平台核心能力建设，包括但不限于 调度、资源管理、控制面组件扩展及平台能力增强。

2. 参与 Kubernetes 控制面关键组件的设计与开发，包括：

Scheduler 调度策略扩展与优化

Controller / Operator 设计与开发

API Server / Admission / CRD 等控制面能力扩展

3. 参与容器云平台核心基础设施建设，包括：

容器运行时与资源管理

多租户隔离与权限控制（RBAC / Auth / Policy）

存储与网络编排能力（CSI / CNI / 网络策略）

4. 参与 AI 训练 / 推理场景下的 Kubernetes 平台能力建设，包括：

GPU / 异构资源调度

大规模训练任务调度优化

资源弹性与调度策略优化

5. 参与云原生平台整体架构设计，持续优化平台 可用性、扩展性和自动化能力，推动云原生最佳实践落地。

任职资格：

1. 计算机相关专业本科及以上学历，5年以上后端研发经验。

2. 至少2年以上 Kubernetes 相关研发经验，深入理解 Kubernetes 核心架构，包括：

API Server

Scheduler

Controller Manager

CRD / Operator 机制

3. 熟练掌握 Golang 开发，具备良好的系统设计与工程能力。

4. 熟悉 Kubernetes 生态组件及原理，包括但不限于：

CNI / CSI / Container Runtime

Kubernetes 调度框架（Scheduling Framework）

Admission Webhook

Operator / Controller 开发

5. 熟悉 Linux 操作系统与基础网络原理，理解容器运行时（containerd / runc）基本机制。

6. 具备良好的问题分析与解决能力，能够在复杂分布式系统中定位问题。

加分项

满足以下任意一项优先：

1. 有 Kubernetes Scheduler / 调度框架开发经验

2. 有 控制面组件开发经验（API Server / Controller / Operator）

3. 有 CSI / CNI 插件开发经验

4. 有 Kubernetes Auth / RBAC / Admission / Policy 相关经验

5. 有 GPU / AI 训练平台调度经验

6. 有 大规模 Kubernetes 集群（3000+ nodes）性能调优经验

7. 参与过 Kubernetes 社区或相关开源项目
